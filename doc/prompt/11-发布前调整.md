# 发布前调整

## 背景

我们已经完成了 ppclaws.com 网站的开发工作，并且准备部署上线。

在正式为用户提供服务之前，我重新审视了 [搜索关键词](../research/openclaw_keyword_mindmap.md) 文档，并决定调整菜单。

从而在保持网站服务特色的前提下，能够覆盖更多的用户搜索关键词。

同时，因为我们有一些菜单的内容还没有准备好，所以我们需要暂时隐藏部分菜单项和页面的内容板块。

经过深思熟虑，我决定将菜单层级及菜单项调整如下：

- index
  - openclaw
    - setup
    - usage
    - agent
    - alternatives

  - Example

    - All examples
  - Community

**中文菜单**

## 任务

我希望你帮我执行下列任务：

- 菜单设计合理性评估
  - 阅读 [搜索关键词](../research/openclaw_keyword_mindmap.md) 文档和 [](../research/openclaw_google_trend.xlsx) 文档
  - 判断我的菜单设计是否合理：是否涵盖了大部分用户主要的搜索意图？同时保持了菜单的精简性？
  - 每个菜单及子菜单的用词是否最佳？是否与用户常用的搜索词相同/相近？或者潜在的可能搜索词？
  - 菜单的英文版本单词大小写是否符合惯例？
- 关联改动
  - 我们调整了菜单项（新增了 openclaw 菜单，社区菜单；修改 skills/example 的子菜单；剔除了 training，submit 菜单）
  - 识别出关联的受影响页面、逻辑、配置文件，看看是否因此需要进行关联的改动
  - 附加说明
    - 由于我们现在没有培训的业务，所以 traning 菜单及对应的板块（Training + FAQ 板块）可以剔除（不要隐藏代码，防止被人查看源代码）
    - 将现在的 training 板块的内容替换为一个调研问卷（用于调研用户对 openclaw 的熟悉情况和内容需求）。你帮我设计问题、界面和交互。
    - 保留现在 submit 表单。虽然没有了一级菜单。当我依然希望鼓励用户提及他们看到的好的 skills、案例。
    - community 菜单被点击时，直接滚动到页面的底部（Footer 区域），让用户看到「Follw us」区域。

## 要求

- 对首页的关联改动先不要直接作用在真正的代码上，先在 [prototype](../prototype/) 目录下生成一个新的首页高保真原型页面
