# 网站内容初始化

## 背景

我们已经完成了 PPClaws.com 网站的搭建和基本配置。接下来我们需要开始为网站添加冷启动内容。

当网站初始化内容被正确添加后，我们将可以看到正确的页面。

## 任务

接下来你将需要阅读 [skills_260406.md](../skills/skills_260406.md) 文件，并执行如下任务：

- Step1：阅读 skills_260406.md 文件，了解每个 skills 的基本情况
- Step2：根据 Markdown 文件构建对应的目录树（首页->二级分类->三级分类->技能详情页）
- Step3：为每一层级的页面，创建对应的 JSON 配置文件，用于配置该页面的元数据和内容
- Step4：为每个技能创建对应的 Markdown 文件，用于展示该技能的详细信息
  - 每个技能的 Markdown 文件名必须与该技能的标题一致，且文件扩展名必须为 .md
  - 每个技能的 Markdown 文件必须包含以下元数据：
    - 标题（title）：技能的标题，用于在网站上展示。和 Markdown 文件名一致，均为技能本身的名称。
    - 描述（description）：根据清单提供的技能 URL，找到该 URL 目录下的 SKILL.md 文档，提取该文档的「description」frontmatter 字段内容。
    - 作者（author）：技能的作者而非网站的作者。根据清单提供的技能 URL，提取 URL 内的作者信息。
    - 日期（date）：技能的添加日期，用于在网站上展示。默认为当前日期（年-月-日）
    - 标签（tags）：默认为 []
    - 分类（category）: 根据文档内技能所属的分类进行提取。可用分类为：
      - 产品经理(product manager)
      - 设计设计师(UI/UX Designer)
      - 市场营销人员(Marketing)
      - 开发人员(Developer)
    - 封面文本（cover_text）：默认为技能的标题
    - 封面主题（cover_theme）：默认为 "dark-blue"
    - 是否精选（featured）：默认为 false
    - 徽章（badge）:默认为空
    - 相关资源（relatedResources）：默认为：`https://code.claude.com/docs/en/skills`
  - 每个技能的 Markdown 文件内，必须包含如下章节
    - 简介 (H2标题)：该章节内容为该技能 SKILLS.md 的「description」frontmatter 字段内容。
    - 功能（H2标题）：通读该技能的 SKILLS.md 文档后，用 3-5 个列表项总结该技能都做了哪些事情。
    - 亮点（H2标题）：通读该技能的 SKILLS.md 文档后，你要归纳出该技能在设计思想/实现流程上的亮点。
- 所有的 Skills 都按照添加日期的顺序进行展示。越新添加的 Skills 越靠前。
- 如果某个技能在 Excel 文件中被标识为「精选」，则该技能对应的卡片会出现在首页和 Skills 的精选区域。但依然遵循按时间顺序展示的原则。
