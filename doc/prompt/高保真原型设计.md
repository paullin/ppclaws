# 高保真原型设计

## 背景

通过 [竞品分析](./竞品分析.md) 文档，我们对 ppclaws.com 首页的内容板块划分有了基本的了解。

同时通过[竞品调研分析结果](../调研分析结果.md)，我们对竞品 openclaw101.dev 网站也有了一定的研究。

现在我想基于这两份文档，来设计一个高保真的首页原型。

## 任务

### 首页

设计一个全新的 ppclaws.com 首页高保真原型页面。要求：

- 视觉风格完全复刻现在的 Astro 主题的风格
- 网站的名字从模板的 'Starter' 改为 'PPClaws'，字体与字号、颜色不变
- 网站的 Logo 使用 [logo](../../logo.webp) 图片，放置在网站名称左侧，填充合适的间距
- 导航栏的功能按钮与现在保持一样（搜索、暗夜模式、语言切换、Github 链接）
- 导航栏的一级菜单+二级菜单根据 [竞品分析](./竞品分析.md) 文档和 [关键词分析](../openclaw_google_trend.xlsx)，设计出一个符合用户需求的导航结构。
- 导航栏的一级菜单+二级菜单的布局与现在保持一样，只是文字内容不同。
- 内容区域：由以下子板块构成
  -- 轮播图：用于编辑精选的内容，跑马灯效果。区域宽高比为 16:9.
  -- 子区域1：精选 Skills。包括 4 个精美的图文卡片 + 浏览更多的文字链接
  -- 子区域2：Skills for product manager。包括 4 个精美的图文卡片 + 浏览更多的文字链接
  -- 子区域3：Skills for UI/UX designer。包括 4 个精美的图文卡片 + 浏览更多的文字链接
  -- 子区域4：Skills for Marketing。包括 4 个精美的图文卡片 + 浏览更多的文字链接
  -- CTA 按钮：邮件订阅输入框 + 按钮
  -- 子区域5：精选案例。包括 4 个精美的图文卡片 + 浏览更多的文字链接。卡片内部右上角有「free」「paid」标签
  -- 子区域6：提交收录。用于用户提交他们的 Skills、案例、文章等。包括一个表单，用户可以在表单中填写他们的信息，提交后会自动跳转到一个确认页面。
- 顶部区域的参考 [OpenClaw101.dev](https://openclaw101.dev/) 的链接思路（关联资源的链接、作者自身社交媒体账号链接、赞助商链接）

### 子目录

设计二级 + 三级目录页的高保真原型页面。要求(以英文语言版本为例，中文版本请参考英文版本)：

- 二级目录页分别是对应首页导航栏的 skills 和 example 一级子菜单下的 more 菜单项。
- 当用户点击这两个一级菜单下的 more 菜单项时，链接到对应的二级目录页。
- 二级目录页的页面布局与首页保持一致。包括：页面结构、栅格系统、响应式规则。
- 二级目录页的元素视觉风格与首页保持一致。包括尺寸比例、颜色、字体、间距、圆角、边框、阴影、透明度、渐变、图标/图片大小、位置、比例、交互状态（默认/hover/点击/禁用）。
- Skills 二级目录页包括4个子区域：skills for product manager、skills for UI/UX designer、skills for Marketing、skills for Developer。每个区域由 4个精美的图文卡片 + 更多文字链构成
- Example 二级目录页包括5个子区域：requirement analysis、Prototyping、A/B Testing、SEO keyword mining、Content generation。每个区域由 4 个精美的图文卡片构成 + 更多文字链构成
- 当点击每个区域的更多文字链时，会跳转到三级目录页。
- 三级目录页和二级目录页的布局相同，唯一的区别仅仅是三级目录页仅展示单个区域内的所有卡片。例如 skills for product manager 三级目录页仅展示 skills for product manager 这个区域内的所有卡片。
- 三级目录页下的每个卡片都有一个链接，点击链接后会跳转到对应的卡片详情页。
- 每个三级目录页都有返回二级目录页的链接 + 指向兄弟目录页的链接。每个二级目录页都有返回一级目录的链接 + 指向兄弟目录页的链接。
- 最终形成的链接目录结构如下，以英文版本为例：
  -- index
    -- skills  
      -- skills for product manager   # 对应三级目录页 ppclaws.com/skills/skills-for-product-manager.html
      -- skills for UI/UX designer    # 对应三级目录页 ppclaws.com/skills/skills-for-ui-ux-designer.html
      -- skills for Marketing         # 对应三级目录页 ppclaws.com/skills/skills-for-marketing.html
      -- skills for Developer         # 对应三级目录页 ppclaws.com/skills/skills-for-developer.html
      -- more                         # 对应二级目录页 ppclaws.com/skills.html
    -- example
      -- requirement analysis         # 对应三级目录页 ppclaws.com/example/requirement-analysis.html
      -- Prototyping                  # 对应三级目录页 ppclaws.com/example/prototyping.html
      -- A/B Testing                  # 对应三级目录页 ppclaws.com/example/a-b-testing.html
      -- SEO keyword mining           # 对应三级目录页 ppclaws.com/example/seo-keyword-mining.html
      -- Content generation           # 对应三级目录页 ppclaws.com/example/content-generation.html
      -- more                         # 对应二级目录页 ppclaws.com/example.html
- 二级页面原型要全部生成，包括中英文版本。三级页面只需生成「skills-for-product-manager.html」即可，但要包含中英文版本

## 约束

- 请使用 HTML + Tailwind CSS + JS 来实现这个原型。
- 输出请放置在 `../prototype/` 目录下。
