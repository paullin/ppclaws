# SEO 规范检查

以下是一份完整详细的 SEO 规范检查。请你根据项目的实际情况和所处阶段，选择最小必须检查项目。

所有需要上线后或依赖第三方的检查项，都先直接忽略。

并将结果输出到[SEO 规范检查结果](../tech/SEO 规范检查结果.md)

## 一、基础配置检查

### 1. 域名与服务器配置

如果当前网站已经上线，则需检查以下内容。如果网站未上线，则无需检查第1项。

- [ ] 域名解析正常（无DNS解析错误，可通过站长工具验证）
- [ ] 全站启用HTTPS（无HTTP页面，SSL证书有效，无证书警告）
- [ ] 服务器稳定（访问速度正常，无频繁宕机，响应时间≤2s）
- [ ] WWW与非WWW、HTTP与HTTPS统一（设置301重定向，避免域名重复）
- [ ] 域名备案完成（国内服务器必查，无备案风险）

### 2. 抓取与索引基础

- [ ] 页面状态码检查：所有核心页面（首页、分类页、测试内页）返回200 OK
- [ ] 无404、403、500等错误页面（测试所有导航、内链，确保无死链）
- [ ] robots.txt配置正确（放在根目录，无误屏蔽首页、分类页等核心页面）
- [ ] XML Sitemap生成并有效（包含所有核心页面，无错误链接，可正常访问）
- [ ] 无错误noindex标签：`<meta name="robots" content="noindex">`
- [ ] 无robots meta标签误设置（如noarchive、nofollow等，非必要不添加）

## 二、网站结构与内链检查

### 1. 网站层级结构

- [ ] 采用扁平结构：核心页面（首页→分类页→内页）点击深度≤3次
- [ ] 导航清晰：主导航、副导航逻辑合理，覆盖所有核心板块
- [ ] 面包屑导航配置完成（显示当前页面层级，链接可点击，利于抓取和用户体验）
- [ ] 无孤立页面（所有内页都有至少1条内链指向，无“无人问津”的页面）

### 2. URL规范

- [ ] URL简洁、小写，无特殊字符（如@、#、&多余参数）
- [ ] URL语义化（包含页面核心内容，不使用乱码、随机数字）
- [ ] 无重复URL（分页、筛选页无过多冗余参数，避免造成重复内容）
- [ ] URL静态化（优先使用静态URL，动态URL需做伪静态处理）

### 3. 内链配置

- [ ] 内链分布均匀：首页指向所有分类页，分类页指向相关内页，内页之间相互关联
- [ ] 锚文本自然：内链锚文本不堆砌关键词，贴合页面内容
- [ ] 无无效内链（内链指向页面均为200状态，无404、403）
- [ ] 核心页面（如首页、重点分类页）获得更多内链权重支持

## 三、页面技术细节检查

### 1. 标题与层级

- [ ] 每页只有1个H1标签（无多个H1，不堆砌关键词，贴合页面内容）
- [ ] 标题层级规范：H1→H2→H3依次递减，无层级混乱（如H3直接接H1）
- [ ] H2/H3标签合理：用于拆分页面内容，覆盖页面子主题，无滥用

### 2. 图片优化

- [ ] 图片文件名语义化（如`lightweight-travel-backpack-women.jpg`，不使用默认文件名）
- [ ] 所有图片添加ALT文本（描述图片内容，不堆砌关键词，适配图片搜索）
- [ ] 图片压缩处理（体积≤200KB，避免大图片拖慢加载速度）
- [ ] 启用图片懒加载（延迟加载首屏以外的图片，提升页面加载速度）
- [ ] 图片格式合理（优先使用WebP格式，兼顾清晰度和体积）

### 3. 页面基础优化

- [ ] canonical标签正确（核心页面canonical指向自身，无自我冲突、无指向404）
- [ ] 无重复内容（分页、筛选页、相似页面已通过canonical或noindex处理）
- [ ] 页面无大量空白、乱码、格式错乱（PC端+移动端均检查）
- [ ] 无多余弹窗、广告遮挡核心内容（避免影响抓取和用户体验）

## 四、速度与Core Web Vitals检查

### 1. 页面加载速度

- [ ] 使用Google PageSpeed Insights测试，PC端+移动端得分≥80分
- [ ] 启用浏览器缓存（设置合理的缓存时间，减少重复加载）
- [ ] 压缩JS、CSS文件（合并冗余文件，减少文件体积，避免阻塞渲染）
- [ ] 启用CDN加速（尤其针对多地区用户，提升加载速度）
- [ ] 移除无用代码（删除未使用的JS、CSS，减少页面冗余）

### 2. Core Web Vitals核心指标

- [ ] LCP（最大内容绘制）：≤2.5s（优先优化首屏核心内容加载速度）
- [ ] INP（交互下一步延迟）：≤200ms（优化页面交互响应速度，无卡顿）
- [ ] CLS（累积布局偏移）：≤0.1（避免页面加载过程中元素错位、跳动）
- [ ] 无长期阻塞渲染的资源（如大型JS文件、未压缩的图片）

## 五、移动端适配检查

- [ ] 移动端友好（通过Google移动端友好性测试，无适配错误）
- [ ] 移动端排版正常：无文字过小、按钮过密、内容溢出屏幕
- [ ] 移动端加载速度：与PC端差距不大，得分≥70分
- [ ] 移动端导航适配：汉堡菜单可正常展开/关闭，无点击困难
- [ ] 无移动端专属错误（如弹窗遮挡、Flash插件，移动端不支持Flash）

## 六、安全与合规检查

- [ ] 全站HTTPS加密（无混合内容警告，即HTTP资源嵌入HTTPS页面）
- [ ] 无恶意代码、黑链（通过站长工具检测，确保网站干净）
- [ ] 网站后台安全：设置复杂密码，开启二次验证（非必要不对外开放后台）
- [ ] 隐私政策页面（若收集用户信息，需有合规的隐私政策，明确数据使用规则）
- [ ] 无侵权内容（图片、文字等均有授权，避免版权纠纷）

## 七、高级技术配置检查

### 1. 结构化数据（Schema）

- [ ] 首页：配置Organization（组织）结构化数据，展示品牌信息
- [ ] 文章页/产品页：配置对应结构化数据（Article/Product），提升SERP展示效果
- [ ] 面包屑、导航：配置BreadcrumbList结构化数据，帮助Google理解页面层级
- [ ] 结构化数据无错误（通过Google Rich Results Test验证）

### 2. 其他高级配置

- [ ] 分页配置正确（使用rel="next"和rel="prev"标签，避免分页内容重复）
- [ ] JS渲染页面（若使用SPA）：确保Google能正常抓取渲染后的内容
- [ ] 404页面自定义：设计友好的404页面，添加返回首页链接，避免用户流失
- [ ] 301重定向检查：所有需要重定向的页面（如旧URL）均正确设置301，无302跳转
- [ ] 网站地图自动更新：设置Sitemap自动更新，新增/删除页面后及时同步至GSC

## 八、上线前最终核验

- [ ] 重新测试所有核心页面状态码，确保无404、500错误
- [ ] 再次验证robots.txt、Sitemap有效性，确认已提交至GSC
- [ ] 测试Core Web Vitals指标，确保所有核心指标达标
- [ ] 用不同浏览器（Chrome、Firefox、Safari）+ 不同设备（PC、手机、平板）测试访问体验
- [ ] 检查内链、外链，确保无死链、无效链接
- [ ] 确认服务器稳定，无宕机、访问缓慢问题
- [ ] 记录网站初始数据（如GSC抓取量、索引数量），方便上线后对比分析

### 补充说明

1. 检查工具推荐（免费好用）：
    - 页面状态码/死链：Screaming Frog SEO Spider（免费版足够）
    - Core Web Vitals/速度：Google PageSpeed Insights
    - 移动端友好性：Google移动端友好性测试
    - 结构化数据：Google Rich Results Test
    - 抓取/索引：Google Search Console（提前注册绑定域名）
2. 优先级说明：
    - 必查项（一、二、三、四、五部分）：不达标绝对不能上线，会直接影响抓取和排名；
    - 可选推荐项（七部分）：上线后可逐步优化，但提前配置能提升上线后SEO起步速度。
