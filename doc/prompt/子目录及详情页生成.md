# 子目录及详情页生成

## 背景

前面我们已经在 [首页替换方案](../prompt/首页替换方案.md) 中实现了首页替换方案。

接下来我们需要生成二级目录、三级目录的页面。

我已经在 [prototype](../prototype/)目录下实现了原型页面。

整个 ppclaw.com 的导航结构如下（以英文版为例，中文版请参考英文版的例子）：

-- index
    -- skills
        -- skills for product manager   # 对应三级目录页 ppclaws.com/skills/skills-for-product-manager.html
            -- prd automation assistant # 对应卡片详情页 ppclaws.com/skills/prd-automation-assistant.html
        -- skills for UI/UX designer    # 对应三级目录页 ppclaws.com/skills/skills-for-ui-ux-designer.html
        -- skills for Marketing         # 对应三级目录页 ppclaws.com/skills/skills-for-marketing.html
        -- skills for Developer         # 对应三级目录页 ppclaws.com/skills/skills-for-developer.html
        -- more                         # 对应二级目录页 ppclaws.com/skills.html
    -- example
        -- requirement analysis         # 对应三级目录页 ppclaws.com/example/requirement-analysis.html
        -- Prototyping                  # 对应三级目录页 ppclaws.com/example/prototyping.html
        -- A/B Testing                  # 对应三级目录页 ppclaws.com/example/a-b-testing.html
        -- SEO keyword mining           # 对应三级目录页 ppclaws.com/example/seo-keyword-mining.html
        -- Content generation           # 对应三级目录页 ppclaws.com/example/content-generation.html
        -- more                         # 对应二级目录页 ppclaws.com/example.html

这些页面是用 HTML + Tailwind CSS + JS 实现的。

## 任务

现在我要基于这个原型页面来生成相应的二级目录、三级目录、详情的页面框架。

不同于 [首页替换方案](../tech/首页替换方案.md) 中的替换 Astro 主题的首页，这次我们是要从无到有地生成二级目录、三级目录、详情的页面框架。

同时我们要复用新首页的工作成果。包括但不限于

- 页面布局框架
- 页面组件
- 页面样式定义
- 路由规则

同时由于这个 Astro 主题本身就带有一些目录，我不清楚在我们这次生成中，是否会产生冲突？

- [assets](../../src/assets/)
- [components](../../src/components/)
- [blog](../../src/content/blog/)
- [homepage](../../src/content/homepage/)
- [data](../../src/data/)
- [i18n](../../src/i18n/)
- [layouts](../../src/layouts/)
- [locales](../../src/locales/)
- [pages](../../src/pages/)
- [styles](../../src/styles/)
- [utils](../../src/utils/)

请帮我分析如果我们要基于原型页面来生成真正的页面，并满足下面的要求，需要做哪些改动？

- 生成后的页面需要和原型页面保持百分百像素级一致。
- 生成后的页面使用的技术栈需要和当前网站使用的技术栈一致。
- 生成后的二级、三级目录页面所有板块需要可以通过运营后台配置出来（参考首页的可配置化）。
- 生成后的详情页内容（Markdown 格式）需要渲染出和原型页面一致的样式。详情页内容的架构是固定的 [格式示例](../prototype/skills/prd-automation-assistant-cn.html)
- 生成后的页面需要支持响应式布局。
- 原项目下有哪些目录、代码、配置文件因为我们本次改动，可以废弃的？
