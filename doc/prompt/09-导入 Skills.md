# 导入 Skills

## 背景

我们已经完成了 PPClaws.com 网站的搭建和基本配置。接下来我们需要开始为网站添加 Skills。

## 任务

接下来你将需要阅读 [skillslist.txt](../data/skillslist.txt) 文件。该文件的特点如下：

- 按日期划分为多个部分：分隔符的格式是【YYYY-MM-DD】，按日期倒序排列：即文档最顶部的日期永远都是最新一批技能的添加日期。
- 每两个分隔符之间是一批 URL 列表。即我们要导入的技能 URL 列表。格式如下：
  - 类型：可选项为 skills 对应二级分类
  - 领域：可选项为产品经理、设计师、市场营销人员、开发者。对应三级分类
  - URL：技能的 SKILLS.md 文档的 URL。有1-多个，对应技能详情页

### 步骤

- Step1：根据上面的规则，找出最新添加的一批技能 URL
- Step2：根据 Markdown 文件构建对应的目录树（首页->二级分类->三级分类->技能详情页）
- Step3：为每一层级的页面，创建对应的 JSON 配置文件，用于配置该页面的元数据和内容
- Step4：为每个技能创建对应的 Markdown 文件，用于展示该技能的详细信息
  - 数量要求：中文和英文各一份
  - 位置要求：严格按照 Astro 的要求，放置到对应语言目录下
  - 文件名要求：中英文文件名均使用技能的名称作为文件名，多个单词之间使用连线符「-」连接，不得使用中文拼音或其他非英文字符。
  - 元数据要求：
    - 标题（title）：技能的标题，和 Markdown 文件名一致，均为技能本身的名称。
    - 描述（description）：根据每个技能 URL 目录下的 SKILL.md 文档，提取该文档 frontmatter 中的 description 字段内容。
    - 作者（author）：从 Github URL 内提取技能作者的名称
    - 日期（date）：技能的添加日期，默认为当前日期（年-月-日 时:分）
    - 标签（tags）：默认为 []
    - 封面主题（cover_theme）：遵循如下约定：
      - 产品经理技能：默认为 "dark-blue"
      - 设计设计师技能：默认为 "dark-green"
      - 市场营销人员技能：默认为 "dark-orange"
      - 开发人员技能：默认为 "dark-purple"
    - 是否精选（featured）：默认为 false
  - 正文要求：**严格遵循** `../doc/prompt/10-Skills 解析.md` 文档的要求。
  - 翻译要求：优先导入英文版本技能介绍，再将所有章节的内容翻译为中文。要求：语言通俗易懂，减少专业术语的使用，专注用户的需求。
  - 排序要求：所有的 Skills 都按照添加的时间顺序进行展示。越新添加的 Skills 越靠前。
- Step6：更新首页/二级目录/三级目录的 JSON 配置文件，确保新添加的内容卡片能够在对应的页面上展示
- Step7：运行 npm run build 命令
- Step8：运行 npm run dev 命令
