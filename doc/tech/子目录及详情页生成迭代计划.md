# 子目录及详情页生成迭代计划

根据《子目录及详情页生成方案》，为了便于开发与验收，整个方案划分为以下五个小的迭代任务：

## 迭代一：历史代码清理与依赖配置

- **本次迭代的目标**：清理项目中因新方案而废弃的旧有路由、代码与配置，并引入必要的样式依赖，避免新旧路由冲突。
- **本次迭代的改动范围**：
  - **删除废弃路由与页面**：删除 `src/pages/` 目录下的 `[about]/`, `[blog]/`, `[pages]/`, `[dyn_routing]/`, `[pagination]/`, `[...blog].astro`, `[...contact].astro`。
  - **删除废弃数据集合**：删除 `src/content/blog/` 及 `src/content/authors/`。
  - **清理旧有组件与布局**：删除 `src/components/` 和 `src/layouts/` 中仅服务于旧博客的组件（如旧版 BlogCard、Pagination、AuthorCard）。
  - **修改数据配置文件**：在 `src/content.config.ts` 中移除与 `blog` 和 `authors` 相关的 Schema 定义。
  - **修改 Tailwind 配置**：在 `tailwind.config.js` 中引入 `@tailwindcss/typography` 插件。
- **本次迭代的验收方式**：执行 `npm run build` 能够成功构建无报错，旧版路由访问返回 404，原系统正常运行。

## 迭代二：内容集合（Content Collections）结构定义

- **本次迭代的目标**：定义 Astro 内容集合的数据结构，为二三级目录及详情页提供配置化数据支持。
- **本次迭代的改动范围**：
  - **修改数据配置文件**：在 `src/content.config.ts` 或 `src/content/config.ts` 中新增 `skillsCategory` 数据集合的 Schema（包含 id, title, description, sections）。
  - **新增详情页 Schema**：在相同的配置文件中新增 `skills` 和 `example` 内容集合的 Schema（包含 title, description, author, date, tags, category, coverImage, badge, relatedResources, recommendedReading 等元数据）。
  - **新增测试数据**：在 `src/content/` 目录下创建基础的 JSON/YAML 目录配置文件和少量 Markdown 格式的详情页示例。
- **本次迭代的验收方式**：项目的 TypeScript 类型检查通过，Astro 开发服务器启动后能正确读取新定义的集合数据且无类型报错。

## 迭代三：公共组件与全局 Markdown 样式开发

- **本次迭代的目标**：开发详情页与目录页所需的专用 UI 组件，并配置全局排版样式，确保视觉效果与高保真原型完全一致。
- **本次迭代的改动范围**：
  - **新增基础组件**：在 `src/components/` 下新建 `Breadcrumbs.astro`（面包屑）、`ArticleHeader.astro`（详情页顶部区域）、`RelatedLinks.astro`（关联资源列表）及 `ExploreMore.astro`（探索其他分类卡片模块）。
  - **修改全局样式**：在全局 CSS 文件（如 `src/styles/global.css`）的 `@layer base` 层中自定义 `.prose` 样式，如覆盖 `.prose h2` 的字体大小和间距，以解决 Tailwind 默认样式优先级冲突问题。
- **本次迭代的验收方式**：在一个测试页面中引入上述新组件，结合 Tailwind 样式和自定义的 `.prose` 类，组件能够正常渲染且排版、间距、字体等视觉细节严格符合原型标准。

## 迭代四：二、三级目录页路由与功能实现

- **本次迭代的目标**：实现二、三级目录的动态路由、国际化（i18n）支持及页面模块化渲染。
- **本次迭代的改动范围**：
  - **新增二级目录页面**：创建 `src/pages/[...lang]/skills/index.astro` 及 `src/pages/[...lang]/example/index.astro`。
  - **新增三级目录页面**：创建 `src/pages/[...lang]/skills/[category].astro` 及 `src/pages/[...lang]/example/[category].astro`。
  - **组件集成**：在上述页面中复用 `Base.astro`，整合 `SectionHeader.astro`, `SectionGrid.astro`, `SkillCard.astro` 等组件，并接入 Content Collections 提供的数据。
- **本次迭代的验收方式**：通过浏览器访问二级目录（如 `/skills` 或 `/zh/skills`）与三级目录（如 `/skills/product-manager`），页面能正确展示对应的分类标题及精选卡片列表，且多端响应式布局正常。

## 迭代五：详情页动态路由与 Markdown 渲染实现

- **本次迭代的目标**：实现最终的详情页动态路由，并将 Markdown/MDX 内容完美转化为符合原型设计的网页。
- **本次迭代的改动范围**：
  - **新增详情页路由**：创建 `src/pages/[...lang]/skills/[category]/[slug].astro` 及 `src/pages/[...lang]/example/[category]/[slug].astro`。
  - **页面组装与渲染**：在页面中整合 `Breadcrumbs.astro` 和 `ArticleHeader.astro`；使用 `<article class="prose dark:prose-invert max-w-none">` 包装 Astro 的 `<Content />` 组件以渲染主体 Markdown；在页面底部加入 `RelatedLinks.astro` 和 `ExploreMore.astro`。
- **本次迭代的验收方式**：访问具体的详情页路由（如 `/skills/product-manager/prd-automation-assistant`），页面能够完整展示上下游导航与相关推荐，Markdown 内容（包含标题、列表、引用等）格式渲染正确，深浅色模式切换表现符合预期。
