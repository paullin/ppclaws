# 发布前调整方案评估

## 1. 菜单设计合理性评估

经过对搜索关键词思维导图（`openclaw_keyword_mindmap.md`）和 Google Trend 意图分类数据（`openclaw_google_trend.xlsx`）的对比分析，得出以下评估结论：

### 1.1 意图覆盖与精简性
- **意图覆盖**：目前的菜单设计非常合理。Google Trend 数据显示用户的核心搜索意图集中在“相关信息”、“安装部署”、“使用教程”、“渠道接入”、“竞品对比”和“技能扩展”等方面。新菜单中的 `openclaw`（包含安装、使用、Agent、竞品）和 `Skills`、`Example` 完美覆盖了这些核心意图。新增的 `Community` 也满足了用户寻找交流渠道（如微信、Telegram等）和周边项目的需求。
- **精简性**：将具体的安装、配置、教程、竞品等细分意图收敛在 `openclaw` 一级菜单下，极大地保持了顶部导航栏的精简，降低了用户的认知负荷。

### 1.2 用词准确性与潜在搜索词
- **用词契合度**：子菜单用词（如 setup/安装, usage/使用, alternatives/竞品）与用户常用的搜索词（install, use, alternative）高度一致。
- **优化建议**：英文菜单中的 `Example` 建议改为复数形式 `Examples`，以更符合英语习惯。

### 1.3 英文版本大小写规范
当前的英文菜单大小写存在不统一的情况（部分全小写，部分句首大写）。根据英文网站导航栏的常见惯例（Title Case），建议统一调整如下：
- `openclaw` -> `OpenClaw`
- `setup` -> `Setup`
- `usage` -> `Usage`
- `agent` -> `Agent`
- `alternatives` -> `Alternatives`
- `Skills` -> `Skills` (保持不变)
  - `For product manager` -> `For Product Manager`
  - `For UI/UX designer` -> `For UI/UX Designer`
  - `For marketing` -> `For Marketing`
  - `For developer` -> `For Developer`
  - `All skills` -> `All Skills`
- `Example` -> `Examples`
  - `Brainstorming` -> `Brainstorming`
  - `writing PRD` -> `Writing PRD`
  - `Prototyping` -> `Prototyping`
  - `Data analysis` -> `Data Analysis`
  - `SEO` -> `SEO`
  - `All examples` -> `All Examples`
- `Community` -> `Community`

## 2. 关联改动评估

本次菜单调整及板块替换将影响以下代码和配置：

### 2.1 页面与组件 (Pages & Components)
1. **导航栏组件**：
   - 需更新顶部导航数据源，新增 `OpenClaw` 和 `Community` 菜单，重构 `Skills` 和 `Examples` 的下拉子菜单。
   - 移除一级菜单中的 `Training` 和 `Submit`。
2. **首页内容 (Index Page)**：
   - **移除**：删除原有的 `Training` 课程推广板块和 `FAQ` 手风琴问答板块。
   - **新增**：在原 `Training` 位置新增“用户调研问卷（Survey）”板块，用于收集用户对 OpenClaw 的熟悉程度及内容需求。
   - **保留**：保留 `Submit`（收录提交）表单板块，以鼓励用户共建。
3. **Footer 区域**：
   - 为 Footer 中的 “Follow Us” 区域添加锚点 ID（例如 `id="follow-us"`），以便 `Community` 菜单点击时平滑滚动至此。

### 2.2 配置文件 (Configurations)
1. **多语言文件 (`src/locales/en/common.json` & `zh/common.json`)**：
   - 需要新增新菜单项的 key-value 映射（如 `menu.list.openclaw`, `menu.list.setup`, `menu.list.community` 等）。
   - 更新或新增问卷板块的相关文案。
2. **路由与数据 (`src/data/navigationData.ts`)**：
   - 按照新的菜单层级结构完全重写 `navigationData` 数组。
3. **内容配置 (`src/content.config.ts` 等)**：
   - 如果首页内容由 Astro Content Collections 驱动，需将 `training` 和 `faq` 的 schema 定义移除或改为 `survey` 相关字段。

### 2.3 逻辑与交互 (Logic & Interactions)
1. **锚点滚动**：确保点击 `Community` 时触发 `document.getElementById('follow-us').scrollIntoView({behavior: 'smooth'})`。
2. **问卷表单交互**：需要设计单选/多选/文本输入等题型，以及“提交成功”的反馈状态。
