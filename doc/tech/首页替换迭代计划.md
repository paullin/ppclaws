# PPClaws 首页替换迭代计划

根据 [首页替换方案](file:///Users/paullin/Dropbox/Study/Project/Python/ppclaws/doc/tech/首页替换方案.md) 的要求，现将实施过程划分为以下 7 个迭代阶段，以确保开发进度可控且便于验收。

---

## 迭代 1：全局样式与基础布局适配 (Infrastructure & Layout)

- **本次迭代的目标**：搭建符合高保真原型的 Tailwind v4 全局样式体系与基础 Layout 架构。
- **本次迭代的改动范围**：
  - **修改** [global.css](file:///Users/paullin/Dropbox/Study/Project/Python/ppclaws/src/styles/global.css)：引入原型定义的颜色变量、字体系统及 Tailwind v4 新特性。
  - **修改** [Base.astro](file:///Users/paullin/Dropbox/Study/Project/Python/ppclaws/src/layouts/Base.astro)：调整 HTML 结构，优化暗夜模式切换逻辑（适配 `dark:` 类名与 `localStorage` 持久化）。
  - **修改** [content.config.ts](file:///Users/paullin/Dropbox/Study/Project/Python/ppclaws/src/content.config.ts)：为后续 Content Collections 预留首页 Schema 定义。
- **本次迭代的验收方式**：
  - 运行项目，检查全局背景色、文字颜色是否与原型一致。
  - 手动切换系统/手动暗夜模式，验证页面背景与文字是否能正确响应 `.dark` 类。

## 迭代 2：页眉与多语言导航 (Header & Navigation)

- **本次迭代的目标**：实现具备粘性毛玻璃效果的高保真 Header 及响应式导航。
- **本次迭代的改动范围**：
  - **修改** [Header.astro](file:///Users/paullin/Dropbox/Study/Project/Python/ppclaws/src/components/Header.astro)：重构为原型中的设计，实现 `backdrop-blur` 效果、多级下拉菜单。
  - **修改** [LanguagePicker.astro](file:///Users/paullin/Dropbox/Study/Project/Python/ppclaws/src/components/LanguagePicker.astro)：整合至 Header 内部，确保中英文切换逻辑正常。
  - **修改** [navigationData.ts](file:///Users/paullin/Dropbox/Study/Project/Python/ppclaws/src/data/navigationData.ts)：更新导航数据结构以匹配原型分类。
- **本次迭代的验收方式**：
  - 滚动页面，验证 Header 是否固定在顶部并带有毛玻璃半透明效果。
  - 在桌面端和移动端测试导航菜单的展开与收起。
  - 点击语言切换，验证 URL 及 Header 文案是否同步更新。

## 迭代 3：首屏轮播交互 (Hero Carousel)

- **本次迭代的目标**：使用 Svelte 实现高性能的响应式首屏轮播图。
- **本次迭代的改动范围**：
  - **新增** `src/components/HeroCarousel.svelte`：基于 Svelte 5 实现，支持自动播放、指示器切换、触控滑动。
  - **修改** `src/pages/[...index].astro`：在首页引入 HeroCarousel 组件。
  - **新增** `src/content/homepage/home.json`：存储轮播图的图片路径、标题及跳转链接。
- **本次迭代的验收方式**：
  - 验证轮播图在 16:9 (移动端) 和 21:9 (桌面端) 比例下均无拉伸变形。
  - 检查自动播放与手动点击指示器是否冲突。

## 迭代 4：内容板块与标准化组件 (Section & Grid Components)

- **本次迭代的目标**：开发可复用的内容网格、板块标题及技能卡片组件。
- **本次迭代的改动范围**：
  - **新增** `src/components/SectionHeader.astro`：支持标题、副标题及“浏览更多”链接。
  - **新增** `src/components/SectionGrid.astro`：实现 1/2/4 列自适应的 CSS Grid 布局。
  - **新增** `src/components/SkillCard.astro`：实现卡片的悬停缩放、阴影效果及暗夜模式适配。
- **本次迭代的验收方式**：
  - 在测试页面放置不同列数的 Grid，验证响应式断点（sm/md/lg/xl）切换是否平滑。
  - 悬停在卡片上，检查缩放动画是否符合原型预期。

## 迭代 5：精选案例与状态标签 (Featured Cases & Status Badges)

- **本次迭代的目标**：实现带有“免费/付费”标签的特色案例卡片展示。
- **本次迭代的改动范围**：
  - **新增** `src/components/CaseCard.astro`：继承 `SkillCard` 样式，并增加状态标签（Badge）逻辑。
  - **修改** `src/pages/[...index].astro`：在首页引入特色案例板块，支持 4 列自适应布局。
- **本次迭代的验收方式**：
  - 验证卡片右上角的标签是否根据数据正确显示“Free”或“Paid”。
  - 检查卡片在悬停时是否与 SkillCard 保持一致的缩放效果。

## 迭代 6：培训课程板块 (Training Section)

- **本次迭代的目标**：实现原型中的“Training”板块，包含响应式左右分栏布局及功能清单。
- **本次迭代的改动范围**：
  - **新增** `src/components/TrainingSection.astro`：实现 1:1 左右分栏，左侧图片/占位符，右侧课程详情及 Checklist。
  - **修改** `src/pages/[...index].astro`：在首页引入培训板块。
- **本次迭代的验收方式**：
  - 验证在移动端该板块是否自动转为垂直堆叠布局。
  - 检查 Enroll Now 按钮的悬停动画是否符合原型设计。

## 迭代 7：FAQ 手风琴交互 (FAQ Accordion)

- **本次迭代的目标**：使用 Svelte 5 实现可折叠的常见问题解答（FAQ）板块。
- **本次迭代的改动范围**：
  - **新增** `src/components/FAQ.svelte`：实现平滑的手风琴效果，支持单选/多选展开。
  - **修改** `src/pages/[...index].astro`：在首页引入 FAQ 板块。
- **本次迭代的验收方式**：
  - 点击问题标题，验证内容是否平滑展开/收起，且右侧 Chevron 图标方向随之改变。

## 迭代 8：订阅与资源提交表单 UI (Form UI Components)

- **本次迭代的目标**：完成邮件订阅（Newsletter）区域及资源提交表单的前端 UI 开发。
- **本次迭代的改动范围**：
  - **新增** `src/components/Newsletter.astro`：实现居中对齐的订阅表单区域，包含输入框与订阅按钮。
  - **修改** `src/components/SubmissionForm.svelte`：完成表单的所有输入项 UI（名称、类型、链接、描述、联系方式）。
- **本次迭代的验收方式**：
  - 检查订阅区域在移动端的内边距（Padding）适配。
  - 验证提交表单的所有 Select 和 Textarea 样式是否与原型一致。

## 迭代 9：全局页脚与社交媒体 (Footer & Social Links)

- **本次迭代的目标**：实现包含多列导航、社交媒体图标及版权信息的全局 Footer。
- **本次迭代的改动范围**：
  - **新增** `src/components/Footer.astro`：重构页脚，支持 4 列响应式布局，整合 Lucide 图标。
  - **修改** `src/layouts/Base.astro`：将旧版页脚替换为新组件。
- **本次迭代的验收方式**：
  - 检查社交图标（X, YouTube, WeChat 等）在悬停时是否有颜色变化。
  - 验证底部版权信息在不同屏幕宽度下的对齐情况。

## 迭代 10：内容管理配置化 (Operational Configuration)

- **本次迭代的目标**：将首页**所有**板块（包括新增的 5 个板块）的文案与图片配置从代码中抽离。
- **本次迭代的改动范围**：
  - **修改** [content.config.ts](file:///Users/paullin/Dropbox/Study/Project/Python/ppclaws/src/content.config.ts)：完善 `homepage` 集合的 Schema 定义，包含 FAQ、Training、Footer 等数据结构。
  - **修改** `src/content/homepage/zh.json` / `en.json`：填充完整的所有板块中英文数据。
  - **修改** `src/pages/[...index].astro`：通过 `getCollection` 获取完整数据并注入全页组件。
- **本次迭代的验收方式**：
  - 修改 JSON 中的 FAQ 题目，刷新页面验证无需修改代码即可生效。
  - 切换语言，验证全页（包括新增的 5 个板块）内容均能正确展示对应语言。

## 11. 迭代 11：表单交互与 D1 数据采集 (Forms & Backend API)

- **本次迭代的目标**：实现邮箱订阅与资源收录表单的交互逻辑，并将数据存入 Cloudflare D1。
- **本次迭代的改动范围**：
  - **修改** `src/components/Newsletter.astro`：接入 API 提交逻辑。
  - **修改** `src/components/SubmissionForm.svelte`：接入 API 提交逻辑，添加加载状态。
  - **新增** `src/pages/api/subscribe.ts` / `submit.ts`：Astro API 路由，对接 D1 数据库。
  - **修改** `wrangler.toml` (若存在)：配置 D1 数据库绑定。
- **本次迭代的验收方式**：
  - 提交空表单或错误邮箱，验证前端实时校验报错。
  - 成功提交后，验证数据库中能查看到新增记录。

## 12. 迭代 12：搜索集成与最终调优 (Search & Final Polish)

- **本次迭代的目标**：完成全站搜索集成并进行像素级视觉还原校对。
- **本次迭代的改动范围**：
  - **修改** `astro.config.mjs`：配置 `astro-pagefind` 确保新首页内容被索引。
  - **修改** `src/styles/global.css`：进行最后的细节微调（间距、字体平滑、过渡动画）。
  - **全面测试**：针对主流浏览器（Chrome/Safari/Firefox）及移动设备进行兼容性检查。
- **本次迭代的验收方式**：
  - 在 Header 搜索框输入关键词，验证能否搜到新首页中的卡片内容。
  - 使用原型对比工具，确保最终页面与设计稿误差在 2px 以内。
