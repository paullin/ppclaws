# 子目录及详情页生成方案

## 背景与目标

前面已经在 [首页替换方案](首页替换方案.md) 中确定了 Astro 首页的替换路线。本次任务需要根据 `doc/prototype` 目录下的高保真原型页面，生成 PPClaws 网站的二级目录（如 Skills 技能中心）、三级目录（如产品经理 Skills）及对应的详情页（如 PRD 自动化助手）。

**核心要求**：

- 百分百像素级还原原型。
- 沿用当前技术栈（Astro v5 + Tailwind CSS v4 + Svelte v5）。
- 二级、三级目录支持通过运营后台配置化（类似首页机制）。
- 详情页 Markdown 内容能正确渲染出原型的样式（排版、引用、列表等）。
- 支持响应式布局。
- 清理项目中因本次改动而废弃的目录、代码与配置。

## 页面架构与路由方案

基于 Astro 的动态路由机制与目前的国际化（i18n）架构，我们建议按以下结构规划路由，同时支持中英文（默认 `en`，中文加 `/zh/` 前缀）：

- **二级目录页**：路由为 `[...lang]/skills/index.astro` 及 `[...lang]/example/index.astro`。主要功能是展示对应频道下所有子分类（如 PM、UX）的精选卡片。
- **三级目录页**：路由为 `[...lang]/skills/[category].astro` 及 `[...lang]/example/[category].astro`。主要功能是展示单一子分类（如 `skills-for-product-manager`）下的所有内容。
- **详情页**：路由为 `[...lang]/skills/[category]/[slug].astro` 及 `[...lang]/example/[category]/[slug].astro`。主要功能是渲染具体的 Markdown/MDX 页面内容（如 `prd-automation-assistant`）。

## 运营配置化实现 (Content Collections)

为满足“通过运营后台配置”的要求，并且兼容基于 Markdown 的详情页，需在 `src/content/` 目录下扩展现有的集合（Collections）：

- **目录配置数据 (JSON/YAML)**：在 `src/content/config.ts` 中新增数据集合，用于配置二级/三级目录的页面板块。例如定义 `skillsCategory`（类型为 `data`），Schema 包含 `id`、`title`、`description`，以及 `sections` 数组（用于配置板块标题、副标题及关联的具体 Skill slug 列表）。
- **详情页内容 (Markdown)**：新增 `skills` 和 `example` 的 Content Collection（类型为 `content`），Schema 定义需覆盖原型中的 Metadata。例如包含 `title`、`description`、`author`（默认 PPClaw 团队）、`date`、`tags`、`category`（所属三级目录）、`coverImage`、`badge`（如"推荐工具"）、`relatedResources` 及 `recommendedReading`。

## 组件复用与开发

为了实现高效开发与一致性体验，需最大限度复用已在首页中开发的组件，并补充详情页专用组件：

- **复用已有组件**：沿用 `Base.astro` 作为页面框架（包含 Header、Footer、暗夜模式及 SEO）。同时复用 `SectionHeader.astro`、`SectionGrid.astro`、`SkillCard.astro` 用于渲染二级和三级目录中的卡片列表。
- **新增页面组件**：
  - `Breadcrumbs.astro`：面包屑导航（如 `首页 > Skills 技能中心 > 产品经理 Skills`）。
  - `ArticleHeader.astro`：详情页顶部区域（16:9 大图 + 标题 + 徽章 + 作者/时间/标签 Meta 信息）。
  - `RelatedLinks.astro`：用于渲染详情页底部的“关联资源”与“推荐阅读”列表。
  - `ExploreMore.astro`：三级目录底部的“探索其他分类”卡片模块。

## 样式与 Markdown 渲染（像素级还原）

原型详情页的内容通过 `<article class="prose">` 容器进行了排版。在 Astro 中，需要进行以下处理以确保视觉一致性：

- **引入 Typography 插件**：在 `tailwind.config.js` 中引入 `@tailwindcss/typography` 插件以支持 Markdown 内容的基础样式重置。
- **自定义全局 Prose 样式**：为了严格匹配原型，避免全局选择器与 Tailwind 实用类的特异性（Specificity）冲突，需在全局 CSS 文件的 `@layer base` 中覆盖默认样式。例如将 `.prose h2` 的字体设为 2xl 并加粗，调整段落间距和文字颜色等。
- **Markdown 渲染包装**：通过 `<article class="prose dark:prose-invert max-w-none">` 包裹 Astro 的 `<Content />` 组件，使其应用上文定义的排版样式。
- **响应式适配**：严格沿用原型代码中设定的 Tailwind 断点（如 `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4`），确保从移动端到桌面端的平滑过渡，无需额外编写媒体查询代码。

## 历史包袱清理 (废弃分析)

由于我们采用了全新的配置化首页、目录页与详情页，原 Astro 主题中附带的大量默认模板与路由将发生冲突且失去作用。以下目录、代码与配置文件可以被**安全废弃/删除**：

- **可废弃的路由与页面**：`src/pages/` 目录下所有原主题的动态路由与演示页面，包括 `[about]/`、`[blog]/`、`[pages]/`、`[dyn_routing]/`、`[pagination]/`、`[...blog].astro`、`[...contact].astro`。如果新首页替换方案已采用确定的语言路由方案，原 `[...index].astro` 也可根据情况重构或替换。
- **可废弃的内容集合**：`src/content/blog/` 及 `src/content/authors/` 已不再需要，将被我们的 `skills` 和 `example` 集合替代。
- **可废弃的组件与布局**：`src/layouts/` 及 `src/components/` 下所有仅服务于上述废弃页面（如旧版 BlogCard、旧版 Pagination、AuthorCard 等）的组件均可清理。
- **配置文件调整**：检查 `src/content.config.ts`，移除与 `blog` 和 `authors` 相关的 Schema 定义。
