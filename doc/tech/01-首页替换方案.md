# 首页替换方案分析报告

## 1. 背景概述

PPClaws 网站目前基于 Astro 框架开发，支持多语言 (i18n)、Tailwind CSS 和搜索功能。现需要将当前首页替换为高保真原型页面，并确保改动后的页面满足像素级还原、运营配置化、数据采集、多语言支持及暗夜模式等核心要求。

## 2. 技术栈对齐与集成方案

当前网站技术栈为 **Astro v5 + Tailwind CSS v4 + Svelte v5**，原型页面使用的技术栈与之高度一致，集成方案如下：

- **基础架构**：沿用 Astro 的静态站点生成能力，确保 SEO 友好。
- **UI 框架**：使用 Tailwind CSS 进行样式实现。原型中的 Lucide 图标可集成 `lucide-svelte` 或直接内联 SVG。
- **交互组件**：轮播图 (Carousel) 和表单 (Forms) 等具有交互状态的模块采用 Svelte 实现，以获得更好的性能与用户体验。

## 3. 页面架构与组件化拆分

为实现像素级还原且便于维护，将原型页面拆分为以下 Astro/Svelte 组件：

- **Layouts**:
  - `Base.astro`: 统领全局样式、SEO 元数据及暗夜模式逻辑。
- **Navigation**:
  - `Header.astro`: 实现原型中的粘性毛玻璃效果 (Backdrop Blur)、多级下拉菜单、搜索入口、中英文切换及 GitHub 链接。
- **Home Sections**:
  - `HeroCarousel.svelte`: 16:9/21:9 比例的响应式轮播图，支持自动播放与触控切换。
  - `SectionHeader.astro`: 统一的板块标题组件，支持“浏览更多”链接。
  - `SkillCard.astro`: 技能卡片组件，支持悬停缩放效果及暗夜模式适配。
  - `SectionGrid.astro`: 灵活的网格布局组件（1/2/4 列自适应）。
- **Forms**:
  - `SubscriptionForm.svelte`: 邮箱订阅表单。
  - `SubmissionForm.svelte`: 资源收录表单。

## 4. 运营配置化实现 (Admin Backend)

为满足“通过运营后台配置”的要求，建议采用 **Git-based CMS (如 Decap CMS 或 TinaCMS)** 结合 Astro 的 **Content Collections**。

- **数据存储**：
  - 在 `src/content/homepage/` 目录下存储 `zh.json` 和 `en.json`。
  - 定义 `homepage` 集合的 Schema，涵盖轮播图数组、板块列表、文案、图片路径及跳转链接。
- **Schema 定义**：

  ```typescript
  // src/content.config.ts 示例
  const homepage = defineCollection({
    loader: file("src/content/homepage/home.json"),
    schema: z.object({
      hero: z.array(z.object({ image: z.string(), title: z.string(), link: z.string() })),
      sections: z.array(z.object({
        id: z.string(),
        title: z.string(),
        subtitle: z.string(),
        items: z.array(z.object({ title: z.string(), desc: z.string(), cover: z.string() }))
      }))
    })
  });
  ```

- **运营操作**：管理员通过 CMS 后端界面修改 JSON 内容并提交，触发 Cloudflare Pages 自动构建发布。

## 5. 数据采集与存储方案

由于站点托管在 Cloudflare Pages，数据存储方案如下：

- **接口实现**：在 `src/pages/api/` 目录下创建 Astro API 路由（适配 Cloudflare Functions）。
- **存储引擎**：
  - **Cloudflare D1 (SQL)**: 存储结构化的收录信息与订阅邮箱。
  - **Cloudflare KV**: 存储简单的配置或计数器。
- **流程控制**：
  - 前端 Svelte 表单异步提交至 API 路由。
  - API 验证数据后写入 D1 数据库。
  - 支持简单的管理页面或直接通过 Cloudflare 控制台导出 CSV 数据。

## 6. 核心功能适配

- **多语言支持 (i18n)**：利用现有的 `src/i18n/` 逻辑，通过 `main.json` 或新增的 `homepage` 集合实现内容翻译。
- **搜索功能**：通过 `astro-pagefind` 自动索引新页面内容。
- **暗夜模式**：
  - 原型已包含 `dark:` 类名，需确保全局 `theme-toggle` 逻辑能正确切换 HTML 节点的 `.dark` 类。
  - 使用 `localStorage` 持久化用户选择。
- **响应式布局**：严格执行原型中的 Tailwind 断点（`sm`, `md`, `lg`, `xl`），确保各设备体验一致。

## 7. 后续实施步骤建议

1. **基础组件开发**：将原型中的 HTML 静态代码迁移为 Astro/Svelte 组件。
2. **数据模型构建**：定义 Content Collections Schema，并将文案从 HTML 抽离至 JSON。
3. **后端接口对接**：配置 Cloudflare D1 数据库，编写表单提交 API。
4. **i18n 与 暗夜模式调试**：确保语言切换与主题切换在全页生效。
5. **像素级校对**：通过浏览器对比工具进行视觉校对。
