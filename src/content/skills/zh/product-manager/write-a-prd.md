## 简介

通过用户访谈、代码库探索和模块设计创建 PRD，然后将其作为 GitHub issue 提交。当用户想要编写 PRD、创建产品需求文档或计划新功能时使用。

## 功能

- 通过用户访谈和代码库探索，梳理待解决的问题和潜在方案
- 引导用户进行深度模块化设计（Deep Modules），提升代码的可测试性
- 提供标准化的 PRD 模板（包含问题陈述、解决方案、用户故事等）
- 支持将编写好的 PRD 作为 GitHub Issue 提交

## 亮点

- 强调深层模块化思维，将产品需求与代码实现紧密结合
- 鼓励对用户方案的各种分支和依赖进行严密的质询（relentlessly interview）
- 使用标准化的用户故事格式（As an actor, I want... so that...）规范需求描述
