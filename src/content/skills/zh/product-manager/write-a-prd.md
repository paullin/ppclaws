## 简介

通过用户访谈、代码库探索和模块设计创建 PRD，然后作为 GitHub Issue 提交。当用户想要编写 PRD、创建产品需求文档或计划新功能时使用。

## 功能

- 进行用户访谈以收集需求
- 探索代码库以使功能与现有架构保持一致
- 设计模块化的产品需求
- 自动将最终的 PRD 作为 GitHub Issue 提交

## 适用

最适合在启动需要正式文档和跨团队协作的主要功能或史诗任务时使用。

## 亮点

与 GitHub Issues 无缝集成，将对话式的需求收集直接转化为可操作的项目管理工单。

## 使用

当您需要起草产品需求文档时调用此技能。它将引导您回答问题并输出结构化的工单。
