## Introduction

Create a PRD through user interview, codebase exploration, and module design, then submit as a GitHub issue. Use when user wants to write a PRD, create a product requirements document, or plan a new feature.

## Features

- Conducts user interviews to gather requirements
- Explores the codebase to align features with existing architecture
- Designs modular product requirements
- Automatically submits the final PRD as a GitHub issue

## Applicability

Best used when kicking off a major feature or epic that requires formal documentation and cross-team alignment.

## Highlights

Seamlessly integrates with GitHub issues, turning conversational requirements gathering directly into actionable project management tickets.

## Usage

Call this skill when you need to draft a Product Requirements Document. It will guide you through questions and output a structured issue.
