## Introduction

Create a PRD through user interview, codebase exploration, and module design, then submit as a GitHub issue. Use when user wants to write a PRD, create a product requirements document, or plan a new feature.

## Features

- Explores the problem space and potential solutions through user interviews and codebase exploration.
- Guides users toward deep modular design to improve code testability.
- Provides a standardized PRD template (including Problem Statement, Solution, and User Stories).
- Supports submitting the written PRD as a GitHub Issue.

## Highlights

- Emphasizes deep modular thinking, tightly coupling product requirements with code implementation.
- Encourages relentless interviewing to thoroughly question branches and dependencies in the user's plan.
- Uses standardized user story formats (As an actor, I want... so that...) to specify requirements.
