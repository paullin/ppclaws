## Introduction and Core Concept
The `onboarding-cro` skill turns the AI into an expert in User Activation and Conversion Rate Optimization (CRO). It tackles the frustrating problem of "users signing up but not sticking around" by systematically redesigning the post-signup experience to drive users toward their "Aha! moment" as quickly as possible.

## Core Concept
Minimize time-to-value and drive user activation. By focusing on interactive "doing" over passive "learning" and defining a clear "Aha! moment," this skill ensures new users build long-term retention habits during their critical first session.

## Workflow Analysis (SOP)
[Read original SKILL.md](https://github.com/coreyhaines31/marketingskills/blob/main/skills/onboarding-cro/SKILL.md)

- **Initial Assessment**: The AI checks existing marketing context to avoid redundant questions, then evaluates the product type (B2B vs B2C), current drop-off points, and explicitly asks the user to define the "Aha! moment" (the action that indicates a user "gets it").
- **Aha! Moment Definition**: Forces the user to identify the specific action that correlates most strongly with retention (e.g., "Create first project + add team member") before designing any UI flows.
- **Flow Design & Triage**: The AI evaluates the immediate post-signup flow (first 30 seconds) against three approaches (Product-first, Guided setup, Value-first) and selects the one with the least friction.
- **Empty State Optimization**: Transforms "Empty States" from dead ends into active onboarding opportunities, ensuring every blank page explains its value and provides a clear primary action.

## Skill Design Evaluation
- **Strengths of SOP Integration**: Requires identifying a specific retention-correlated action (the "Aha! moment") before designing UI flows. Provides structured approaches for handling empty states to drive user activation.
- **Potential Limitations**: Relies on the user having sufficient historical data to accurately identify which actions actually correlate with long-term retention.
- **What Makes a Good Skill**: Codifies UI/UX best practices, such as prioritizing interactive tasks over passive tutorials and applying constraints to onboarding checklists.
