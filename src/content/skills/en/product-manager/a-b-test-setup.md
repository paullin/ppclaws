## Introduction

When the user wants to plan, design, or implement an A/B test or experiment. Also use when the user mentions "A/B test," "split test," "experiment," "test this change," "variant copy," "multivariate test," "hypothesis," "should I test this," "which version is better," "test two versions," "statistical significance," or "how long should I run this test." Use this whenever someone is comparing two approaches and wants to measure which performs better. For tracking implementation, see analytics-tracking. For page-level conversion optimization, see page-cro.

## Features

- Provides guidance on planning, designing, and implementing A/B tests and experiments.
- Starts from a hypothesis, defining specific expected outcomes and data-based predictions.
- Emphasizes single-variable testing (Test One Thing) to ensure clear test results.
- Introduces statistical rigor (predetermining sample size, avoiding early test termination).

## Highlights

- Uses a data-driven hypothesis validation approach to avoid subjective assumptions.
- Backed by rigorous statistics, preventing the statistical error of "peeking and stopping early".
- Proactively looks for existing product marketing context to reduce repetitive questioning.
