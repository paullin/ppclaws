## Introduction

Use when you have a spec or requirements for a multi-step task, before touching code

## Features

- Translates specifications for multi-step tasks into detailed implementation plans.
- Plans file structures and responsibility allocation, clearly defining files to create or modify.
- Breaks down the implementation plan into bite-sized tasks (2-5 minutes per step).
- Integrates engineering principles like DRY, YAGNI, and TDD to guide development.

## Highlights

- Uses extremely granular task breakdowns, such as "write failing test" and "run to ensure it fails".
- Assumes a "zero context" baseline to ensure the generated plan is comprehensive and lacks blind spots.
- Reinforces single responsibility and clear boundaries in file architecture design.
