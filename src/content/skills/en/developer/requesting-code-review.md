## Introduction

Use when completing tasks, implementing major features, or before merging to verify work meets requirements.

## Features

- Automates the code review request process
- Verifies that work meets the initial requirements
- Identifies edge cases and potential bugs before merging
- Ensures coding standards and project conventions are followed

## Applicability

Essential for engineering teams and solo developers who want a rigorous final check before merging code into the main branch.

## Highlights

Acts as an automated peer reviewer, catching common mistakes and architectural flaws that might be missed during manual review.

## Usage

Invoke this skill right before you create a Pull Request or consider a major feature 'done'.
