## Introduction

Use when implementing any feature or bugfix, before writing implementation code. Write test first, watch it fail, write minimal code, refactor.

## Features

- Enforces the Red-Green-Refactor cycle
- Writes failing tests before implementation code
- Ensures high test coverage from the start
- Validates minimal required code for features

## Applicability

Perfect for software engineers committed to TDD methodologies or teams wanting to improve their codebase reliability and test coverage.

## Highlights

Strictly gates the writing of production code behind the creation of a failing test, preventing untested logic from entering the codebase.

## Usage

Invoke this skill before writing a new feature. It will first write a test, verify that it fails, and only then write the implementation.
