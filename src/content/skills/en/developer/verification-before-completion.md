## Introduction

Use when about to claim work is complete, fixed, or passing, before committing or creating PRs - requires running verification commands and confirming output before making any success claims.

## Features

- Mandates execution of verification commands (tests, builds, linters)
- Prevents premature claims of success
- Requires undeniable evidence before committing code
- Checks for side effects and regressions

## Applicability

A must-have for all developers to prevent the classic 'it works on my machine' or 'I think I fixed it' scenarios.

## Highlights

Instills a culture of 'evidence before assertions', ensuring that AI assistants never lie about a task being completed without proof.

## Usage

Use this skill when you believe a task is done. It will run the test suite and build processes to mathematically prove the fix.
